# Four-Layer Funnel — 选股四层漏斗

从候选池到核心仓位的系统化筛选流程。每一层都是独立的"淘汰赛"——不过就出局。

---

## Layer 1: Supply Chain Position（产业链位置）

### Core Question

> "如果这个主题的需求翻倍，这家公司的订单是否**确定性**翻倍？"

- Yes, 物理上绕不过去 → 通过
- Maybe, 取决于竞争/客户选择 → 降级观察
- No, 只是蹭了个概念 → 淘汰

### 判断方法

1. **画出需求到供给的物理链路**：终端需求 → 系统集成 → 核心部件 → 原材料
2. **越靠近"不可替代的物理瓶颈"，确定性越高**
3. **问"如果换一家供应商，需要多长时间？"** — 认证周期越长 = 壁垒越高

### 位置分级

| 位置 | 确定性 | 典型公司 | 特征 |
|------|--------|----------|------|
| 原材料/上游 | ★★★★★ | CCL(生益), 光纤(亨通), 石英(菲利华) | 无论谁做下游都用我的料 |
| 核心部件 | ★★★★ | PCB(沪电/深南), 存储(兆易/澜起), 探针卡(强一) | 认证壁垒高,替换成本大 |
| 设备/工具 | ★★★★ | 刻蚀(中微), 光刻(上海微电子) | 国产替代+扩产必需 |
| 系统集成 | ★★★ | 服务器(曙光/浪潮), 液冷(英维克) | 收入大但利润薄,客户有议价权 |
| 应用/软件 | ★★ | 大模型公司, SaaS | 需求确定但变现路径不清晰 |

### 快速判别法

- ✓ "AI 服务器物理上**必须**用这个材料/部件"
- ✓ "全球只有 3-5 家供应商能做"
- ✓ "下游在扩产 → 对上游的采购量线性增加"
- ✗ "公司说自己在做 AI 相关业务"
- ✗ "公司持股了一家 AI 公司"
- ✗ "互动易上回复投资者说有 AI 布局"

## Layer 2: Earnings Validation（业绩验证）

### Core Question

> "最新一期财报是否证明了产业逻辑已经落地？"

### 达标标准（四项至少满足三项）

| 指标 | 及格线 | 优秀线 | 计算方式 |
|------|--------|--------|----------|
| 营收增速 | >30% YoY | >60% | 当季同比 |
| 利润增速 | >营收增速 | >100% | 归母净利同比 |
| 毛利率 | 同比持平或上升 | 上升>2pct | 当季 vs 去年同期 |
| 收现比 | >0.8 | >1.0 | 经营现金流 / 净利润 |

### 关键解读规则

**利润增速 > 营收增速 = 经营杠杆正向放大**
- 含义：收入增长时，固定成本被摊薄，利润弹性放大
- 信号：供不应求 + 有涨价权 + 规模效应显现

**利润增速 < 营收增速 = 增收不增利**
- 含义：在"赔本赚吆喝"或者成本端失控
- 信号：竞争加剧 / 原材料涨价吃掉利润 / 管理费用失控
- 处理：**直接淘汰**，无论叙事多性感

**收现比 < 0.8 = 赚了纸面利润但钱没回来**
- 含义：应收账款堆积，客户付款能力/意愿有问题
- 风险：未来可能出现大额坏账减值
- 处理：降级，除非有合理解释（如政府项目结算周期）

### 时间窗口

| 报告期 | 发布时间 | 行动 |
|--------|----------|------|
| 一季报 | 4月15-30日 | **最关键验证点** — 第一个反映新需求的完整季度 |
| 半年报 | 7-8月 | 趋势确认 |
| 三季报 | 10月15-31日 | 判断全年能否达预期 |
| 年报+Q1快报 | 次年4月 | 新周期起点 |

**最佳买入窗口 = 季报发布后 1 周内**：逻辑验证 + 股价尚未充分反映。

## Layer 3: Valuation（估值）

### Core Question

> "市场给的价格相对于增速来说贵不贵？"

### PEG 为核心尺子

```
PEG = PE(TTM 或 Forward) / 利润增速(%)
```

| PEG | 含义 | 操作 |
|-----|------|------|
| <0.5 | 严重低估 | 重仓候选 |
| 0.5-1.0 | 合理偏低 | 核心仓候选 |
| 1.0-2.0 | 合理 | 可参与但非首选 |
| >2.0 | 高估 | **不推荐** |

### PEG 使用注意事项

1. **用 Forward PE（预期）而非 TTM**：高增长公司 TTM PE 系统性偏高
2. **增速取可持续增速**：Q1 爆发 700% 不等于全年 700%，要看行业景气周期长度
3. **同板块内横向比较**：半导体 PEG 1.0 和消费 PEG 1.0 含义完全不同
4. **PEG 失效场景**：亏损转盈利（用 PS）、周期拐点（用 PB + ROE 回升）

### 估值辅助指标

| 场景 | 用什么 | 阈值 |
|------|--------|------|
| 正常盈利公司 | PEG | <1 |
| 亏损/微利 | PS(市销率) + 毛利率趋势 | PS<5 且毛利率上升 |
| 周期股（有色/化工） | PB + ROE拐点 | PB<2 且 ROE 连续两季回升 |
| 新股（<1年） | PE vs 同行 + 增速差 | PE 低于同行中位数 |

## Layer 4: Flow Confirmation（资金确认）

### Core Question

> "聪明钱是否已经在进场？"

### 板块层面

| 信号 | 含义 | 判断 |
|------|------|------|
| 板块主力净流入 >10亿 | 资金系统性流入 | ✓ 板块 beta 向上 |
| 板块主力净流出 >10亿 | 资金系统性撤退 | ✗ 不逆 beta 做多 |
| 板块内多只票同时异动 | 共识形成中 | ✓ 主线确认 |

### 个股层面

| 信号 | 操作含义 |
|------|----------|
| 缩量新高（换手<3%） | 最强信号 — 没人卖，共识一致 |
| 底部放量不跌 | 有人在低位吸筹 |
| 主力进 + 散户出 | 健康吸筹 → 跟进 |
| 主力出 + 散户进 | **分发 → 回避** |
| 涨停 + 换手<5% | 封板坚决，后续大概率继续 |
| 涨停 + 换手>15% | 分歧大，次日可能开板回调 |
| 天量（5倍以上日均）+ 长上影 | 见顶信号 |

### 从候选到建仓的触发条件

```
Trigger A（回踩买入）:
  股价回踩 5/10 日线 + 成交量缩至高点的 50% 以下 + 不破前低
  → 买入 1/3 仓位

Trigger B（突破买入）:
  放量突破前高（量比>2）+ 板块同日净流入
  → 买入 1/3 仓位

Trigger C（确认加仓）:
  持仓已盈利 + 缩量沿 5 日线上行 + 板块持续净流入
  → 加至 2/3 仓位
```

## Complete Workflow（完整流程图）

```
产业链扫描（每月） → 识别 5-10 只候选
       ↓
季报筛选（每季度 4/7/10/次年4月） → 淘汰不达标，剩 2-3 只
       ↓
PEG 排序 → 确定首选 / 次选 / 备选
       ↓
量价跟踪（每周） → 等待 Trigger A/B/C
       ↓
分批建仓 → 设定止损 + 目标位
       ↓
持仓管理：
  - 缩量沿 5MA 上行 → 持有
  - 放量滞涨 → 减半
  - 跌破止损 → 清仓
  - 板块 beta 反转 → 清仓
```
