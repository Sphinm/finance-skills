# 恒生互联网 ETF套牢+利好不涨 Case Study (May 2026)

重仓恒生互联网ETF(513770)/中概互联ETF(513050)后被套13-15%。5月13-14日阿里腾讯Q1业绩利好出来后盘中冲高3.77%但收盘仅+0.47%，随后连跌3天（5/18恒生科技-2.08%）。这是"ETF套牢陷阱"（Pitfall 07）+ "机会成本盲区"（Pitfall 09）+ "利好不涨"（量价信号9）的组合案例。

---

## Setup

- **标的**: 513770(港股互联网ETF华宝) + 513050(中概互联网ETF易方达)
- **分析日期**: 2026-05-18
- **513770净值**: ~0.44元
- **513050净值**: ~1.18元
- **持仓成本（推算）**: 513770约0.50-0.52, 513050约1.36-1.39
- **亏损**: -13% ~ -15%
- **恒生科技指数**: 4838点，连跌3天，-2.08%

## 为什么被套

### 宏观层面
1. **美债10Y利率上行**（4月CPI超预期）→ 成长股估值系统性承压
2. **中东地缘风险** → 全球risk-off
3. 恒生科技 = f(美债利率40% + 中国经济30% + 盈利30%)，最大权重因素方向不对

### 微观层面
1. **利好不涨 = 有人趁利好出货**：5/14阿里腾讯Q1业绩后盘中+3.77%，收盘仅+0.47%
2. **ETF连续净赎回**：513770连续6日净赎回，累计流出2.36亿
3. **成交缩量**：日均从31.54亿降到21.55亿（-32%），到5/12仅6.04亿
4. **腾讯跌破450港元**创近8个月新低 = 大资金系统性减配港股互联网

## Decision

**结论**: SELL（减半换仓，策略C）

**操作**:
1. 明天减仓50%（接受7-8%的实际亏损）
2. 释放资金转入A股算力主线（生益/亨通/沪电）
3. 保留50%设加仓触发条件：
   - 恒生科技RSI<25 + 连续3天缩量后出现放量大阳线
   - 或513770跌到0.40-0.42（年内低点区域）
4. 止损底线：如果513770跌破0.38 → 全部清仓

**机会成本计算**:
- 等待回本时间: 估计3-6个月（需涨15-18%）
- 同期A股算力预期收益: 20-40%
- 真实成本 = 显性亏损15% + 机会成本30% = **45%**
- 减半换仓后: 显性亏损7.5%（一半仓位锁定）+ 新仓位预期收益20% = **净正向12.5%**

## Lessons

1. **利率环境是港股定价的最大变量**：即使公司业绩好（阿里云+40%），利率方向不对就涨不动
2. **"利好不涨"是最强的见顶/持续弱势信号**：利好都拉不动说明有人在出货
3. **ETF套牢最大的敌人是"等回本"心态**：每多等一天都在支付机会成本
4. **连续净赎回 = 大资金投票离场**：不要跟大资金对着干
5. **跨市场配置需要关注"美债利率对港股估值的压制"这个核心变量**
